System Caddy detection uses an optional caddy executable, explicit/env/common candidate Caddyfile paths, route text checks, and caddy validate. It does not mutate operator-owned files while detecting. The repository starter `infra/caddy/Caddyfile` is not treated as active system Caddy unless the operator explicitly provides that path. Route detection requires the full HappyTG route surface before validation can mark a file reusable.
