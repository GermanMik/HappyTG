Docker mode now has a service strategy step. Reuse is recommended when setup detects reusable services or configured endpoints; non-interactive mode defaults to isolated unless --docker-services reuse is explicit. The installer selects reuse/isolated before port preflight, skips remaps for services that Compose will not start, then builds the final Docker/Caddy plan after port preflight so snippets and patches use remapped host ports. Reuse mode passes COMPOSE_REDIS_URL, COMPOSE_DATABASE_URL, and COMPOSE_S3_ENDPOINT to containers after loopback translation.
