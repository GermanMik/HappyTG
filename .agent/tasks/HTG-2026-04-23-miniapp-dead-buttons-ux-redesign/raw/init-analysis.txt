Initial conclusion to prove/falsify:
- The strongest leading hypothesis is wrong public/private browser API origin derivation.
- If the public HTTPS Mini App injects http://localhost:4000 into browser JS, auth bootstrap and any browser POST action will target the wrong origin.
- Silent catch blocks make the auth-pending screen look like a dead button/dead app instead of showing an actionable error.
