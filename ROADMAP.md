# Engineering Roadmap

HappyTG intentionally separates functional scope from delivery order. The complete capability map is in [docs/engineering-blueprint.md](./docs/engineering-blueprint.md). This file tracks execution-oriented epics.

## Epics

1. Repository bootstrap and monorepo scaffolding.
2. Typed protocol contracts and event taxonomy.
3. Control plane API, worker, and state projections.
4. Telegram bot render layer and pairing flows.
5. Host daemon transport, reconnect, and execution queue.
6. Codex-first runtime adapter.
7. Repo proof engine and task bundle lifecycle.
8. Approval and policy engine integration.
9. Bootstrap and doctor subsystem.
10. Mini App inspection and artifact views.
11. Observability and audit pipeline.
12. Security hardening and threat validation.
13. Documentation pack and contributor onboarding.
14. Self-hosted packaging and upgrade flow.
15. QA, release, and smoke suites.

## Done Criteria

An epic is done only when:

- contracts are documented,
- verification commands are reproducible,
- state transitions are resumable,
- failures are observable,
- degraded mode behavior is defined.
