# HappyTG 0.3.1

## Release version

- `0.3.1`

## Why this release

- Make the one-command onboarding flow resilient when repo connectivity or command launch failures happen at runtime.
- Keep Telegram-first onboarding state across retries so installer reruns behave like resume instead of restart.
- Harden Windows package-manager shim execution and retro TUI paste behavior without replacing the existing bootstrap installer architecture.

## Fixed

- Installer runtime errors no longer fall through to the generic CLI `Usage:` banner; interactive runs stay inside installer-native screens and `--json` / `--non-interactive` runs return structured runtime failures.
- Repo sync now retries transient primary remote failures exactly 5 times with visible progress, then switches automatically to the configured fallback source before giving up.
- Structured installer failures now distinguish repo connectivity exhaustion, fallback-source failures, and command spawn / Windows shim failures, including attempted binary path and suggested action.
- `pnpm install` now reports actionable runtime failures and recovers from safe Windows shim normalization/fallback execution paths.
- Telegram setup input now accepts pasted multi-character chunks cleanly, including long bot tokens, while preserving raw-mode editing and navigation behavior.

## Improved resume behavior

- Installer draft state is persisted under the existing HappyTG local state directory so repo mode, repo directory, repo source, Telegram values, background mode, and post-check selections survive a failed run.
- Subsequent reruns reuse the saved installer draft automatically, allowing non-interactive retries to continue without re-entering Telegram configuration.
- `.env` merge behavior remains idempotent and preserves existing user-owned values.

## Verification

- Added installer-focused tests for repo retry/fallback behavior, structured runtime failures, Windows shim normalization, resume persistence, and Telegram paste handling.
- Release validated with `pnpm lint`, `pnpm typecheck`, `pnpm test`, `pnpm happytg doctor`, `pnpm happytg verify`, and `pnpm release:check --version 0.3.1`.

## Upgrade notes

- Update all workspace versions to `0.3.1`.
- Prefer the existing one-command path: `pnpm happytg install`, `scripts/install/install.sh`, or `scripts/install/install.ps1`.
- If repo access still fails after the automatic fallback attempt, review the structured installer error for the failing source and suggested action before retrying.
