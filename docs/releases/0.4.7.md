# HappyTG 0.4.7

HappyTG `0.4.7` releases the post-0.4.6 control-plane and bootstrap cleanup work.

## Highlights

- `0.4.7`
- Telegram bot session cards now include `Остановить` for active sessions, with API-backed cancellation through `POST /api/v1/sessions/:id/cancel`.
- Session cancellation records the state transition, session event, audit record, and control-plane dispatch cancellation.
- Codex Desktop sessions are now surfaced as first-class control targets in the Bot/Mini App flow where the runtime supports the requested action.
- Docker installer launch mode preflights published Caddy, Prometheus, and Grafana host ports and gives concrete override guidance for bind conflicts.
- Existing `.env` confirmation includes optional Telegram values instead of hiding them when the bot token still needs input.
- `pnpm happytg uninstall` removes HappyTG-owned launcher artifacts and local runtime state while intentionally keeping the repository checkout.

## Upgrade Notes

- No migration is required.
- Existing sessions keep their current state; only active sessions expose cancellation.
- Uninstall cleanup removes only artifacts recorded as HappyTG-owned or safe local state. Repository checkouts are kept by design.
- Docker users with occupied ports can keep using `HAPPYTG_HTTP_PORT`, `HAPPYTG_HTTPS_PORT`, `HAPPYTG_PROMETHEUS_PORT`, and `HAPPYTG_GRAFANA_PORT` overrides.
