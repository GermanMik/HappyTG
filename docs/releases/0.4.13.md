# HappyTG 0.4.13

HappyTG `0.4.13` releases the doctor/verify warning cleanup for successful Codex HTTP fallback and local service reuse diagnostics.

## Highlights

- `0.4.13`
- `pnpm happytg doctor` and `pnpm happytg verify` now stay green when Codex returns the expected smoke `OK` reply after Responses websocket 403 fallback to HTTP.
- Codex smoke checks run from a neutral directory to avoid readiness probes being shaped by repository or global agent instructions.
- Already-running HappyTG services remain visible as `Reuse` guidance, but are no longer emitted as a finding.

## Safety Notes

- Real Codex smoke failures still report `CODEX_SMOKE_FAILED`.
- Higher-level policy, approval ordering, host mutation serialization, Telegram transport boundaries, and Mini App auth behavior are unchanged.
- No migration is required.
