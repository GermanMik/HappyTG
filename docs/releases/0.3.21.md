# HappyTG 0.3.21

## Release version

- `0.3.21`

## Why this release

- Ship the installer repo-sync fix that removes the dependency on checking out the requested local branch during `update/current` flows.
- Make HappyTG installs resilient on repositories that already use linked worktrees, especially when `main` or another requested branch is already occupied elsewhere.
- Preserve the existing dirty-worktree safeguards without forcing users into manual branch cleanup just to update an installer-managed checkout.

## Fixed

- `packages/bootstrap/src/install/repo.ts` now fetches the requested branch, resolves `FETCH_HEAD^{commit}`, and detaches to that fetched commit instead of checking out the local target branch and pulling it.
- Installer update/current flows no longer fail with `fatal: '<branch>' is already used by worktree` just because the branch is already checked out in another linked worktree.
- `packages/bootstrap/src/install.runtime.test.ts` now includes a regression proving the shared sync path updates an existing checkout through the fetched commit without local branch checkout.

## Changed

- The canonical source proof bundle for this release is `HTG-2026-04-19-installer-remote-ref-sync`, and this publication adds a dedicated release bundle for the `0.3.21` metadata/publish path.
