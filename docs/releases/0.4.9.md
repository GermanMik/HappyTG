# HappyTG 0.4.9

HappyTG `0.4.9` releases the local update/uninstall prompt package and synchronized day-2 operator documentation.

## Highlights

- `0.4.9`
- Added `.agent/prompts/local-update-uninstall-user-flow.md` for future agents implementing or auditing easy local update and uninstall UX.
- The prompt includes critical review from 10 independent roles covering Windows users, local developers, Docker operators, system-service reuse, execution-host cleanup, data safety, security, release management, QA, and mobile/APK expectations.
- README, quickstart, installation, bootstrap, troubleshooting, self-hosting, and runbook docs now share the same update guidance: rerun `pnpm happytg install` for the guided path, or use `git pull --ff-only`, `pnpm install`, `doctor`, and `verify` for a clean-checkout manual path.
- Uninstall docs now consistently state that `pnpm happytg uninstall` keeps the repo checkout, `.env`, Docker services/volumes, reused external services, and remote control-plane data.
- APK creation is explicitly gated on a real Android packaging surface. The current repository does not contain Gradle, AndroidManifest, Capacitor, or APK artifacts, so this release does not include a phone-installable APK.

## Upgrade Notes

- No data migration is required.
- Existing local users can update with `pnpm happytg install` followed by `pnpm happytg verify`.
- Docker users should rebuild the selected Compose stack after updating.
- Reused Redis/Postgres/MinIO/Caddy services remain operator-owned and are not removed by HappyTG uninstall.
