# HappyTG 0.4.9

HappyTG `0.4.9` releases the local update command, update/uninstall prompt package, and synchronized day-2 operator documentation.

## Highlights

- `0.4.9`
- Added `pnpm happytg update`, which defaults to the current checkout, skips background launcher changes, skips runtime launch, and runs update post-checks.
- Added `.agent/prompts/local-update-uninstall-user-flow.md` for future agents implementing or auditing easy local update and uninstall UX.
- The prompt includes critical review from 10 independent roles covering Windows users, local developers, Docker operators, system-service reuse, execution-host cleanup, data safety, security, release management, QA, and Telegram/mobile expectations.
- README, quickstart, installation, bootstrap, troubleshooting, self-hosting, and runbook docs now share the same update guidance: run `pnpm happytg update` for the guided path, or use `git pull --ff-only`, `pnpm install`, `doctor`, and `verify` for a clean-checkout manual path.
- Uninstall docs now consistently state that `pnpm happytg uninstall` keeps the repo checkout, `.env`, Docker services/volumes, reused external services, and remote control-plane data.

## Upgrade Notes

- No data migration is required.
- Existing local users can update with `pnpm happytg update`.
- Docker users should rebuild the selected Compose stack after updating.
- Reused Redis/Postgres/MinIO/Caddy services remain operator-owned and are not removed by HappyTG uninstall.
