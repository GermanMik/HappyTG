# HappyTG 0.3.6

## Release version

- `0.3.6`

## Why this release

- Close the remaining Windows installer gap from the real interactive run, where Telegram token and allowed-user-id paste still broke when the terminal delivered one pasted chunk with trailing CRLF.
- Make first-run guidance more accurate about supported non-Docker infra paths so Redis and shared-service follow-up does not read as Docker-only when existing services are already supported through env configuration.

## Fixed

- Telegram setup now normalizes terminal paste chunks with trailing newline / CRLF before commit, preserving masked bot-token previews and existing reducer navigation behavior.
- Allowed user ID paste now accepts at least one pasted numeric ID in the real interactive keypress path, and newline-separated pasted IDs normalize cleanly into the stored list.
- Redis findings and setup guidance now explicitly mention existing-service reuse through `REDIS_URL`, plus broader shared-service reuse through `DATABASE_URL` and `S3_ENDPOINT`, instead of treating Docker Compose as the only path.
- Existing 0.3.5 final-summary aggregation behavior stays intact: warning-level post-check follow-up still dedupes and remains at `success-with-warnings`.

## Verification

- Added regression coverage for:
  - Telegram reducer handling of pasted token chunks with trailing CRLF
  - interactive `promptTelegramForm` paste flow for bot token plus allowed user IDs
  - Redis setup/doctor wording that names existing-service alternatives alongside Compose guidance
- Release validated with:
  - `pnpm --filter @happytg/bootstrap test`
  - `pnpm --filter @happytg/bootstrap typecheck`
  - `pnpm typecheck`
  - `pnpm test`
  - `pnpm build`
  - `pnpm release:check --version 0.3.6`

## Upgrade notes

- Update all workspace versions to `0.3.6`.
- On Windows, terminal paste now works for Telegram token and allowed-user-id fields even when the pasted chunk carries a trailing newline; raw OS clipboard shortcuts still depend on terminal-level paste support rather than a TTY clipboard API.
