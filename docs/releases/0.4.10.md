# HappyTG 0.4.10

HappyTG `0.4.10` releases the Codex Desktop / Codex CLI vision and control repair.

## Highlights

- `0.4.10`
- Codex Desktop visibility stays available through sanitized read-only local state projection for projects, sessions, status, and details.
- Codex Desktop Resume, Stop, and New Task are now unsupported by default because the discovered `codex app-server` and `exec-server` contracts are experimental.
- Unsupported Desktop actions return clear `CODEX_DESKTOP_CONTROL_UNSUPPORTED` reason codes through API responses, audit records, Telegram Bot messages, and Mini App disabled controls.
- Codex CLI task start remains grounded in the proven local `codex exec` contract used by the HappyTG host-daemon flow.
- Codex Desktop mutating actions are classified as serialized host mutations, and injected/proven Desktop control contracts run through the API service mutation queue.

## Upgrade Notes

- No data migration is required.
- Existing Desktop users may still browse Desktop projects and sessions through HappyTG.
- Desktop control buttons intentionally remain disabled until a stable Desktop mutating contract is proven.
- Existing CLI task flows continue to use the HappyTG host-daemon and `codex exec` path.
