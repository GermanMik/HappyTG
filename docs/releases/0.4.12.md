# HappyTG 0.4.12

HappyTG `0.4.12` releases the proof-backed Mini App and Telegram Bot usability/design pass.

## Highlights

- `0.4.12`
- Mini App dashboard now starts with a decision-first next-action summary.
- Session cards now use localized attention labels and clearer primary actions.
- Approval detail now exposes a session-scope approval affordance.
- Telegram Bot `/menu` and active-session messages now stay concise while pointing to the next useful action.
- The implementation keeps detailed inspection in the Mini App and keeps Bot chat short and button-driven.

## Safety Notes

- No runtime policy or approval-ordering semantics changed.
- Telegram callback contracts were not changed.
- Unsupported actions still remain explicit and do not look successful.
- Source/runtime discrimination remains visible for Codex Desktop and CLI flows.

## Upgrade Notes

- No migration is required.
- This is a UX and release-metadata release for the already-merged Mini App and Bot usability improvements.
