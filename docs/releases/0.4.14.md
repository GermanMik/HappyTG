# HappyTG 0.4.14

HappyTG `0.4.14` releases the Docker pnpm activation retry fix, Graphify navigation artifacts, the MinIO compose image pin, and a reusable full-project verification/release prompt.

## Highlights

- `0.4.14`
- Docker app image builds now prepare the `packageManager`-pinned `pnpm@10.0.0` through a bounded Corepack retry layer before `pnpm install`, reducing transient registry/TLS failures during self-hosted builds.
- Docker build context now excludes `.env` and `.env.*` files.
- `infra/docker-compose.example.yml` now uses an explicit MinIO release tag instead of `latest`.
- `graphify-out/` and the matching proof bundle provide a checked-in navigation graph for architecture and dependency work.
- Added `docs/prompts/happytg-full-verification-10-role-release.md` for full verification, branch hygiene, evidence capture, memory, PR/merge, and release workflows grounded in 10 independent roles.

## Safety Notes

- Telegram transport boundaries, policy/approval ordering, serialized host mutations, hooks, Mini App auth behavior, and runtime source semantics are unchanged.
- Graphify semantic refreshes remain local-first and LM Studio oriented when semantic extraction is needed; this release does not add cloud Graphify backends or Ollama fallback.
- The MinIO pin improves reproducibility, but broader image-tag review for Postgres, Redis, Caddy, and Node base images remains separate work.
- No migration is required.
