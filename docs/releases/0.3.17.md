# HappyTG 0.3.17

## Release version

- `0.3.17`

## Why this release

- Finish hardening the installer final-summary close path after `0.3.16` by proving the full interactive runtime releases stdin and can return control to the shell cleanly.
- Add explicit CLI-layer regression coverage for `pnpm happytg install`, so the wrapper path is verified instead of being treated as an untested thin passthrough.
- Keep the already-shipped pairing behavior honest while tightening proof around reuse, refresh, and manual fallback branches.

## Fixed

- `packages/bootstrap/src/install.runtime.test.ts` now includes a full interactive installer regression that drives the final summary to `ENTER close`, confirms stdin is paused again, and verifies there is no lingering `keypress` listener after completion.
- Installer pairing finalization regressions remain locked across existing-host refresh, existing-host reuse, and honest manual fallback, including the request-failed branch that still must hand the user back to `pnpm daemon:pair` when automation cannot proceed safely.

## Changed

- `packages/bootstrap/src/cli.ts` now allows optional test-time runtime overrides for `executeHappyTG()`, and `packages/bootstrap/src/cli.test.ts` uses that seam to prove the CLI install wrapper preserves parsed options, bootstrap-check delegation, and the installer result contract.
- The canonical source proof bundle for the final-summary/exit fix is now complete and validated, and this release adds a dedicated publish bundle for the `0.3.17` metadata/update path.
