# HappyTG 0.3.11

## Release version

- `0.3.11`

## Why this release

- Ship the verified bootstrap/install regression fix after `0.3.10` without widening scope beyond the already-proven diff.
- Restore truthful bootstrap diagnostics on Windows when the shell is poisoned by a stale external `NODE_OPTIONS` preload path, while keeping real environment warnings visible.
- Restore stable interactive progress indicators so the active install step renders readably again on Windows terminals.

## Fixed

- `scripts/install/install.ps1` and `scripts/install/install.sh` no longer collapse a broken external preload path into the false message `Node.js 22+ is still not available on PATH` when Node is present. The wrappers now probe Node startup explicitly, classify missing preloads by scope, clear only broken external `NODE_OPTIONS` for bootstrap-owned commands, and still hard-fail truthfully when the missing preload lives inside `HAPPYTG_BOOTSTRAP_DIR` or the selected workspace.
- The bootstrap fix is explicitly based on root cause evidence, not on inventing a repo-managed `undici` preload artifact. HappyTG still does not own `undici-timeout-preload.cjs`, and the failure reproduced before any HappyTG-managed preload existed.
- The interactive installer progress renderer now uses ASCII-safe `>` for the running step and `.` for pending, with per-status row coloring restored so active/completed/warn/error states stay readable instead of degrading into an unstable purple glyph on some Windows terminals.
