# HappyTG 0.4.17

HappyTG `0.4.17` releases a result-first Telegram Mini App session UI for project work.

## Highlights

- `0.4.17`
- Mini App dashboard, session cards, project cards, Codex panel, and Desktop detail now put work result and next actions before technical metadata.
- New task entry supports implementation tasks, implementation questions, and review prompts from the same form while keeping the backend `quick`/`proof` mode contract intact.
- Secondary information such as long paths, state details, counters, history metadata, and project technical details is hidden behind native disclosure blocks.
- Codex Desktop continuation from `0.4.16`, source-aware Desktop/CLI lists, sorting controls, and unsupported-action disabled states are preserved.

## Safety Notes

- Telegram remains a render/approval surface, not the internal transport for agent events.
- Policy, approval, serialized mutation queues, runtime adapters, and backend task contracts were not weakened.
- Raw prompts, long logs, raw JSON payloads, and secrets stay out of the primary Mini App scan path.
- No migration is required.
