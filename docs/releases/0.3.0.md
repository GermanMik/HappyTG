# HappyTG 0.3.0

## Release version

- `0.3.0`

## Why this release

- Turn HappyTG onboarding into a real one-command installer flow for macOS and Windows while preserving Linux support.
- Move first-run Telegram configuration into the installer so setup no longer stalls on a missing bot token.
- Add release-ready repo shims, docs, and verification artifacts for the new bootstrap path.

## Added

- New `happytg install` flow inside `packages/bootstrap` with retro interactive screens for preflight, repo mode, Telegram setup, background mode, progress, and final summary.
- Thin platform shims at `scripts/install/install.sh` and `scripts/install/install.ps1` that clone or update the repo, then hand off to the shared TypeScript installer implementation.
- Installer state persistence under `~/.happytg/state/install-last.json`.

## Improved onboarding

- Repository sync now supports `clone`, `update`, and `current directory` modes with explicit dirty-worktree handling.
- Dependency checks now cover Git, Node.js 22+, pnpm, Codex CLI, and Docker/Desktop with install commands or precise manual guidance per platform.
- `.env` initialization now merges from template safely, preserves existing values, and stores Telegram bot identity for later `/pair <CODE>` instructions.
- Pairing guidance now targets the configured bot directly when `TELEGRAM_BOT_USERNAME` is known.
- Non-interactive install now requires `--telegram-bot-token` instead of silently leaving setup in a broken state.

## Docs

- Updated [README](../../README.md), [Installation](../installation.md), [Quickstart](../quickstart.md), [Bootstrap Doctor](../bootstrap-doctor.md), and [Configuration](../configuration.md) for the one-command install path.
- Added release metadata for `0.3.0` and aligned workspace package versions.

## Verification

- Added installer-focused tests for platform detection, repo mode decisions, dirty worktree defaults, `.env` merge idempotency, and retro renderer hints.
- Release validated with `pnpm lint`, `pnpm typecheck`, `pnpm test`, `pnpm release:check --version 0.3.0`, `pnpm happytg doctor`, `pnpm happytg verify`, and `pnpm build`.

## Upgrade notes

- Update all workspace versions to `0.3.0`.
- Prefer the new one-command path: `curl -fsSL <install.sh> | bash`, `irm <install.ps1> | iex`, or local `pnpm happytg install`.
- If you automate install, pass `--telegram-bot-token` in non-interactive mode so setup and later pairing can target the configured bot immediately.
