# HappyTG 0.4.25

HappyTG `0.4.25` fixes Mini App project views that could hide past Codex Desktop sessions.

## Highlights

- `0.4.25`
- Project-filtered Codex Desktop session views now request a wider initial window of `100` sessions instead of filtering only the default `50`.
- Desktop sessions without a `projectPath` from Codex Desktop are shown in project views with an explicit note instead of being hidden.
- A `Показать до 200 Desktop sessions` action is available when the first `100` Desktop sessions are filled.
- The default unfiltered Desktop session list remains bounded at `50` for fast startup.

## Safety Notes

- No Codex Desktop mutation behavior changed.
- No auth, policy, approval, or transport behavior changed.
- The change is presentation/fetch policy only for Mini App session lists.
