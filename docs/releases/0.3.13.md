# HappyTG 0.3.13

## Release version

- `0.3.13`

## Why this release

- Ship the verified Windows Telegram transport fallback so HappyTG can recover when Node HTTPS/undici cannot reach the Bot API on this host but Windows PowerShell still can.
- Turn the bootstrap Telegram `getMe` PowerShell probe from a warning-only diagnostic into a real success path when it validates the same bot token.
- Prevent the bot runtime from silently losing outbound replies during Windows-only Node transport failures while still reporting real Telegram HTTP/API failures truthfully.

## Fixed

- `packages/bootstrap/src/install/telegram.ts` now treats a successful Windows PowerShell `getMe` follow-up probe as a real validated bot identity instead of degrading that case into a network warning after a Node HTTPS timeout.
- `apps/bot/src/index.ts` now retries `sendMessage` through a Windows PowerShell Bot API call when the Node HTTPS transport throws, and keeps structured logging for the original Node transport failure plus any fallback failure.
- `packages/bootstrap/src/install.test.ts`, `packages/bootstrap/src/install.runtime.test.ts`, and `apps/bot/src/index.test.ts` now lock the validated fallback path and the negative controls for invalid token / Telegram HTTP failures.
