# HappyTG 0.4.23

HappyTG `0.4.23` fixes the Mini App `Desktop sessions unavailable: This operation was aborted` warning on Codex Desktop session lists.

## Highlights

- `0.4.23`
- Mini App Codex Desktop fallback timeout now defaults to `6000ms`, covering current bounded `/api/v1/codex-desktop/sessions?limit=50` latency.
- Raw abort text is normalized to `request timed out after <N>ms` if a Desktop fallback still times out.
- The bounded fallback behavior remains in place for genuinely slow or unavailable Desktop data.

## Safety Notes

- No Codex Desktop adapter API contract was changed.
- No auth, policy, approval, or mutation behavior was changed.
- The fix is limited to Mini App timeout budget and user-facing timeout text.
