# HappyTG 0.3.2

## Release version

- `0.3.2`

## Why this release

- Close the remaining installer UX gaps after 0.3.1 so Telegram-first onboarding behaves predictably during paste-heavy setup and warning-only runtime cases.
- Keep the existing bootstrap installer architecture while making final-state handling explicit, recoverable, and consistent across interactive and structured execution paths.
- Tighten release readiness for the installer TUI without weakening cross-platform bootstrap behavior on Linux, macOS, or Windows.

## Fixed

- Telegram bot token input in the installer TUI now renders as a masked preview that keeps the first 4 and last 4 characters visible while preserving the raw secret in persisted state only.
- Masked preview updates now stay correct for long pasted chunks, backspace edits, repeated edits, field navigation, and short-token edge cases without leaking the full token into summaries or structured output.
- Installer token validation now stops early for missing, incomplete, placeholder, or mistaken `@botname` values and reports an installer-native validation problem instead of drifting into a late runtime failure.
- Final installer state now distinguishes `success`, `success-with-warnings`, `recoverable-failure`, and `fatal-failure`, removing contradictory combinations such as “Install flow is complete” alongside a hard failure badge.
- Warning-only Telegram bot lookup failures now stay warning-only in both interactive TUI and `--json` / `--non-interactive` results.
- Completed runs with failed follow-up steps now report a recoverable partial failure with a suggested action instead of a dead-end final screen.
- Interactive installs now close directly from the TUI final screen on `ENTER close` without printing an extra plain-text summary after the screen already resolved.

## Verification

- Added installer-focused tests for masked token previews, paste/backspace editing, raw draft persistence, short-token masking, warning-only Telegram lookup handling, recoverable partial failures, invalid `@botname` input, and final-screen close behavior.
- Release validated with `pnpm lint`, `pnpm typecheck`, `pnpm test`, `pnpm happytg doctor`, `pnpm happytg verify`, and `pnpm release:check --version 0.3.2`.

## Upgrade notes

- Update all workspace versions to `0.3.2`.
- Keep using the shared bootstrap installer entrypoints: `pnpm happytg install`, `scripts/install/install.sh`, or `scripts/install/install.ps1`.
- If the installer reports a recoverable failure, fix the named follow-up issue and rerun the installer instead of treating the run as a fatal bootstrap break.
