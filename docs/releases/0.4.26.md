# HappyTG 0.4.26

HappyTG `0.4.26` fixes the follow-up Mini App timeout for project-filtered Codex Desktop session lists.

## Highlights

- `0.4.26`
- Project-filtered Codex Desktop views still request the widened `100` session window introduced in `0.4.25`.
- Desktop sessions fetches with `limit >= 100` now use a scoped minimum `10000ms` timeout budget.
- The default unfiltered Desktop session list remains bounded at `50` and keeps the configured/default timeout.

## Safety Notes

- No Codex Desktop mutation behavior changed.
- No auth, policy, approval, or transport behavior changed.
- The change is Mini App fetch policy only for Desktop session list rendering.
