# HappyTG 0.4.15

HappyTG `0.4.15` releases the doctor/verify local-environment warning cleanup for Codex smoke stderr and the public Mini App Caddy identity route.

## Highlights

- `0.4.15`
- `pnpm happytg doctor` and `pnpm happytg verify` now stay green when Codex returns the expected smoke `OK` reply but emits benign memory phase 403 or slow SQLite PRAGMA diagnostics.
- The local BaseDeploy Caddy source route for `happytg.gerta.crazedns.ru` is repaired and reloaded so public `/miniapp` returns HappyTG Mini App identity instead of HealthOS HTML.
- Mini App proxy headers now force the public HappyTG origin through the fallback route, keeping `window.HAPPYTgApiBase = ""` for same-origin browser API calls.
- The proof bundle records reproduction, Caddy probes, 10-role critical review, full repo validation, and final doctor/verify PASS output.

## Safety Notes

- Real Codex smoke failures still report as actionable warnings or failures.
- Generic HTTP 200 responses are still rejected unless they carry HappyTG Mini App identity.
- HealthOS public health routing was rechecked after the Caddy reload and still returns HealthOS API JSON.
- Telegram transport boundaries, policy/approval ordering, serialized host mutations, hooks, and Mini App auth behavior are unchanged.
- No migration is required.
