# HappyTG 0.4.11

HappyTG `0.4.11` adds a reusable execution prompt for a comprehensive 10-role Mini App and Telegram Bot UX optimization pass.

## Highlights

- `0.4.11`
- Added `docs/prompts/happytg-ux-10-role-optimization.md`.
- The prompt requires 10 independent role assessments before implementation decisions:
  - Telegram Bot Operator
  - Mini App Mobile User
  - First-Time User
  - Daily Power User
  - Accessibility Reviewer
  - Telegram Platform Specialist
  - Control-Plane Safety Engineer
  - Information Architect
  - Visual Product Designer
  - QA / Release Verifier
- The prompt keeps the Mini App and Bot responsibilities separate: Bot stays concise and action-first, while Mini App carries deeper inspection and recovery.
- The prompt preserves HappyTG invariants for source/runtime discrimination, unsupported action honesty, policy/approval ordering, serialized mutations, and proof-loop evidence.

## Upgrade Notes

- No runtime migration is required.
- This release does not redesign the UI by itself; it adds the repo-grounded prompt for the next implementation wave.
- Future UX implementation work should start from a fresh `.agent/tasks/<TASK_ID>/` bundle and use the prompt as its execution contract.
