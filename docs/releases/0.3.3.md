# HappyTG 0.3.3

## Release version

- `0.3.3`

## Why this release

- Close the remaining Windows installer/runtime ambiguity after `0.3.2`, where Telegram secondary lookup failures and Codex PATH follow-up issues could still look more severe than the actual machine state.
- Keep the current bootstrap installer architecture intact while making post-install diagnostics more precise and less contradictory on Windows and in structured install output.
- Preserve cross-platform behavior while tightening the distinction between true configuration breakage and recoverable follow-up environment work.

## Fixed

- Telegram installer lookup now classifies missing token, invalid token rejection, recoverable network fetch failures, and other secondary `getMe` issues separately instead of flattening them into a generic lookup error.
- Installer results now preserve already-known Telegram bot usernames from `.env` when live identity lookup fails, so `/pair` guidance can still target the configured bot directly.
- Telegram warning-only lookup failures no longer make the system read as if Telegram were wholly unconfigured when token, allowed user IDs, and bot identity were already known.
- Bootstrap doctor/setup/verify now retry Codex readiness through detected npm wrapper paths and emit a `CODEX_PATH_PENDING` follow-up warning when Codex is runnable but not yet on the plain shell PATH.
- Windows PATH augmentation inside installer bootstrap now uses normalized PATH handling, avoiding casing-related path loss in environments that expose both `Path` and `PATH`.
- Plain-text installer summaries now explicitly show configured Telegram state even when identity lookup only warns.

## Verification

- Added regression coverage for:
  - runnable npm-wrapper Codex PATH follow-up warnings
  - warning-only Telegram `getMe` lookup failures

  - invalid-token versus recoverable network lookup classification

## Upgrade notes

- Update all workspace versions to `0.3.3`.
- If bootstrap reports `CODEX_PATH_PENDING`, treat it as a follow-up shell PATH fix rather than a missing-Codex install.
- If installer Telegram output reports a warning-only identity lookup issue, check network/API reachability before assuming the stored Telegram bot configuration itself is invalid.
