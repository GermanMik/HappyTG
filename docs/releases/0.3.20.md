# HappyTG 0.3.20

## Release version

- `0.3.20`

## Why this release

- Close the remaining product risk where `pnpm install` could emit ignored build-script warnings during install and leave the user between raw pnpm noise and an installer flow that had not yet proved whether the critical runtime toolchain was actually healthy.
- Keep HappyTG's pnpm security posture intact: no automatic build-script approvals, no global pnpm policy writes, and no fake success when the `tsx`/`esbuild` bootstrap path is unusable.
- Make installer guidance version-aware so users are not told to run `pnpm approve-builds` on a runtime that does not actually expose that command.

## Fixed

- `packages/bootstrap/src/install/index.ts` now classifies ignored build-script warnings from `pnpm install`, probes the repo-local `pnpm exec tsx --eval ...` toolchain path, and distinguishes warning-only healthy installs from broken installs that must fail.
- `scripts/install/install.ps1` and `scripts/install/install.sh` now run a bounded bootstrap preflight for the shared installer, normalize ignored-build-scripts warnings into HappyTG-owned messaging, and hand off through `pnpm --silent dlx` so raw pnpm noise no longer desynchronizes the user-facing install flow.
- Installer diagnostics no longer advertise `pnpm approve-builds` on the pinned/runtime `pnpm` path when that command is unavailable.

## Changed

- The canonical source proof bundle for this release is `HTG-2026-04-19-pnpm-build-script-warning-guard`, and this publication adds a dedicated release bundle for the `0.3.20` metadata/publish path.
