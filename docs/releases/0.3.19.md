# HappyTG 0.3.19

## Release version

- `0.3.19`

## Why this release

- Fix the remaining local Telegram bot failure on Windows hosts where `0.3.18` already selected polling correctly but polling control-plane calls still depended on Node HTTPS reaching Telegram Bot API.
- Keep local `/start` and other Telegram commands usable on affected Windows machines without weakening the explicit `auto|polling|webhook` delivery-mode model.
- Preserve truthful degraded behavior when both Node HTTPS and the Windows fallback path cannot reach Telegram.

## Fixed

- `apps/bot/src/index.ts` now retries transport-level Telegram Bot API failures for `deleteWebhook`, `getUpdates`, and `getWebhookInfo` through a Windows PowerShell fallback when Node HTTPS cannot reach `api.telegram.org`.
- Local polling on affected Windows hosts can now become `ready` instead of staying stuck in degraded `fetch failed` state even though Telegram is reachable through another supported transport on the same machine.
- Bot logs and `/ready` now include actionable transport diagnostics when neither Node HTTPS nor the Windows PowerShell Bot API fallback can complete the Telegram request.
