# HappyTG 0.4.21

HappyTG `0.4.21` fixes misleading Codex Desktop Mini App warnings when opening read-only past sessions and verifies the Docker runtime path for Desktop Projects.

## Highlights

- `0.4.21`
- Past Codex Desktop sessions no longer show `Desktop actions may be disabled` as a page-level warning just because Resume/Stop/New Task are unsupported.
- Desktop session detail keeps read-only history visible without a standalone unsupported-action warning section.
- Disabled Desktop action controls still expose their unsupported reason in action context, preserving the existing runtime contract.
- Docker API/Mini App live smoke was run with `infra/docker-compose.codex-desktop.yml`, confirming local and public Projects pages render Codex Desktop projects instead of the empty adapter state.

## Safety Notes

- Telegram remains a render/approval surface, not the internal transport for agent events.
- No Mini App auth, policy, approval, mutation queue, or Codex Desktop runtime-adapter contract was relaxed.
- Docker deployments that need local Codex Desktop project projections must include `infra/docker-compose.codex-desktop.yml`.
