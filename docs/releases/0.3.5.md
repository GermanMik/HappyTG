# HappyTG 0.3.5

## Release version

- `0.3.5`

## Why this release

- Close the remaining Windows installer UX gap after `0.3.4`, where post-check warnings were shown inside progress steps but not promoted into the final summary.
- Keep the installer outcome at `success-with-warnings` while making the final follow-up explicit and non-duplicative.

## Fixed

- Warning-level follow-up from installer post-checks now flows into the final installer summary and structured install result instead of stopping at step-local detail.
- Repeated `CODEX_PATH_PENDING` warnings from `setup`, `doctor`, and `verify` are deduplicated into one final warning and one actionable PATH next step.
- Final summaries now keep Telegram lookup warnings and Windows Codex PATH follow-up visible together, so the user sees both the completed install state and the concrete environment action that remains.
- Plain-text installer output now reflects the same final warning/next-step aggregation as the interactive summary.

## Verification

- Added regression coverage for:
  - warning-only Telegram plus warning-only Codex PATH final-summary aggregation
  - deduped `CODEX_PATH_PENDING` warning and next-step aggregation across `setup` / `doctor` / `verify`
  - real Windows APPDATA wrapper post-check summaries that stay at `success-with-warnings`
- Release validated with:
  - `pnpm --filter @happytg/bootstrap test`
  - `pnpm --filter @happytg/bootstrap typecheck`
  - `pnpm build`
  - `pnpm lint`
  - `pnpm typecheck`
  - `pnpm test`
  - `pnpm release:check --version 0.3.5`

## Upgrade notes

- Update all workspace versions to `0.3.5`.
- If Windows installer post-checks report `CODEX_PATH_PENDING`, expect one deduplicated PATH follow-up in the final summary rather than three repeated warnings.
