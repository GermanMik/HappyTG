# HappyTG 0.3.9

## Release version

- `0.3.9`

## Why this release

- Finish the remaining installer publish-flow follow-up for Telegram transport diagnostics, Codex websocket warning classification, and final summary deduplication.
- Keep real environment warnings visible while making the product explain which problems are genuine local-machine conditions and which were diagnostic blind spots.

## Fixed

- Telegram bot `getMe` warnings no longer stop at a bare Node timeout when HappyTG can safely prove more: on Windows the installer now runs a PowerShell Bot API follow-up probe to distinguish transport-specific Node/curl failures from invalid tokens and from broader Bot API reachability issues.
- Invalid-token follow-up probes now remain classified as invalid token even when the PowerShell HTTP error body is not available, avoiding false `api_error` wording on `401` and `404`.
- Codex smoke warnings now explicitly state when the Responses websocket returned `403 Forbidden` but the CLI fell back to HTTP and completed the smoke request, keeping the warning accurate without overstating it as a blocking failure.
- Final installer next steps no longer repeat `pnpm dev`, duplicate pairing instructions, or repeat the same post-check warning set across `setup`, `doctor`, and `verify`.
