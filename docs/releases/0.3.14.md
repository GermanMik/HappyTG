# HappyTG 0.3.14

## Release version

- `0.3.14`

## Why this release

- Make installer follow-up easier to scan by separating the detected problem from the proposed resolution path.
- Keep the shared finalization/onboarding model truthful across JSON, plain text, and TUI instead of flattening structured remediation into one sentence.
- Lock the renderer behavior with direct regression coverage so future installer warning/conflict wording changes do not collapse back into merged text.

## Fixed

- `packages/bootstrap/src/finalization.ts` now supports optional `solutions` on structured automation/finalization items while preserving legacy flattened previews for compatibility consumers.
- `packages/bootstrap/src/cli.ts` and `packages/bootstrap/src/install/tui.ts` now render remediation as nested bullets under each problem or warning item instead of forcing everything into one line.
- `packages/bootstrap/src/index.ts` and `packages/bootstrap/src/install/index.ts` now emit structured problem + solution data for Codex PATH follow-up, Telegram token blockers, port conflicts, and related install onboarding/finalization paths.
- `packages/bootstrap/src/install.test.ts`, `packages/bootstrap/src/install.runtime.test.ts`, `packages/bootstrap/src/cli.test.ts`, and `packages/bootstrap/src/index.test.ts` now lock the separate problem/solution rendering contract, including a direct TUI warning-item solutions assertion.
