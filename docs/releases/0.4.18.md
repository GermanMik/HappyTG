# HappyTG 0.4.18

HappyTG `0.4.18` releases a resilience fix for slow or unavailable Codex Desktop project and session loading in the Telegram Mini App.

## Highlights

- `0.4.18`
- Mini App Codex, Sessions, Projects, New Task, and Desktop session detail screens now render with bounded fallbacks when Codex Desktop control is slow or temporarily unavailable.
- Partial Desktop load failures are shown as warnings while available CLI or file-backed Desktop data remains visible.
- Codex Desktop control capability checks are cached and deduplicated, reducing repeated startup waits across Mini App navigation.
- Runtime adapter calls for Desktop project listing, session listing, and session detail reads are now bounded by a short control timeout before falling back to local projections where available.

## Safety Notes

- Telegram remains a render/approval surface, not the internal transport for agent events.
- Policy, approval, serialized mutation queues, and task mutation contracts were not weakened.
- Desktop action capabilities still come from the control contract; unavailable control is represented as disabled actions with explicit reasons.
- No migration is required.
