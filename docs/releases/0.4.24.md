# HappyTG 0.4.24

HappyTG `0.4.24` fixes a Mini App auth recovery reload loop on session detail pages.

## Highlights

- `0.4.24`
- Session detail pages that receive API `401` now clear stale Mini App session state before attempting Telegram re-auth.
- The auth bridge expires the `happytg_miniapp_session` cookie on `401` recovery responses.
- Existing auth bridge behavior remains: the user sees a reconnect/retry screen instead of JSON errors.

## Safety Notes

- No Telegram auth contract was changed.
- No Codex Desktop adapter behavior was changed.
- No policy, approval, or mutation behavior was changed.
- The fix is limited to Mini App auth recovery after rejected session tokens.
