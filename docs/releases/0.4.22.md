# HappyTG 0.4.22

HappyTG `0.4.22` fixes Mini App session-detail pages that could return JSON `500 Internal server error` when the backing API rejected a stale or invalid Mini App session with `401`.

## Highlights

- `0.4.22`
- `/session/:id` now renders the existing Mini App auth bridge on API `401` instead of exposing an internal fetch error.
- Legacy `/?screen=session&id=...` entrypoints use the same recovery path.
- The shared HTTP server supports an optional per-app error mapper; existing services keep their previous default `500` behavior unless they opt in.
- POST actions keep their existing JSON/status handling.

## Safety Notes

- No Mini App session authorization rule was relaxed.
- No Telegram initData, token issuance, policy, approval, or mutation contract was changed.
- The fix only changes how Mini App SSR GET pages present an auth failure to the browser.
