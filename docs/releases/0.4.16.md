# HappyTG 0.4.16

HappyTG `0.4.16` releases Codex Desktop session continuation and Mini App ordering controls.

## Highlights

- `0.4.16`
- Existing Codex Desktop sessions can now receive a new follow-up prompt from the Mini App detail screen.
- The new continuation path uses app-server `turn/start` against the existing thread instead of overloading the older `thread/resume` action.
- `codex_desktop_continue` is a dedicated policy/audit action and remains serialized through the existing API Desktop mutation queue.
- Codex lists can be sorted by update time or title, and Desktop history can be displayed oldest-first or newest-first.
- The proof bundle records the 10-role critical review, implementation scope, validation logs, and local doctor/verify environment blockers.

## Safety Notes

- Desktop `Resume`, `Stop`, and `New Desktop Task` retain their existing contracts.
- Continuation is enabled only when the Desktop adapter reports a supported app-server contract; unavailable app-server state stays truthful through structured errors.
- Telegram remains a render/approval surface, not the internal transport for Desktop events.
- Policy evaluation, user scoping, audit records, and serialized mutating host operations are preserved.
- No migration is required.
