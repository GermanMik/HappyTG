# HappyTG 0.4.0

## Release version

- `0.4.0`

## Why this release

- Ship the five-wave HappyTG MVP foundation as one coherent release instead of leaving the work only in proof bundles.
- Close the main product gaps between API, bot, Mini App, session engine, approval/policy logic, proof-loop bundles, and self-hosted operations.
- Preserve the current TypeScript-first monorepo while adding the missing contracts, UX, security, and deployment readiness layers.

## Added

- Canonical foundation contracts for domain entities, states, events, daemon protocol, proof-loop bundle metadata, and Caddy path routing.
- Bot-first UX with Russian action-first menu, guided task wizard, smart defaults, session cards, host/session/approval browsing, and scoped approval callbacks.
- `@happytg/session-engine` for reducer-backed session transitions, resume semantics, and transition tests.
- `@happytg/telegram-kit` for Telegram Mini App `initData` validation and signed `startapp` launch payloads.
- Mini App launch grants, short-lived app sessions, dashboard/session/approval/host/report/diff/verify projections, mobile-first screens, and local draft recovery.
- API `/version`, API `/metrics`, Mini App revoke paths, log redaction, Caddy/Prometheus/Grafana compose scaffolding, and operational runbooks.

## Changed

- Session, approval, policy, tool execution, proof lifecycle, bot, and Mini App flows now evolve the existing HappyTG apps/packages instead of introducing a second architecture.
- Approval resolution is idempotent and nonce-aware.
- Policy evaluation respects scoped policy layers and keeps higher-level denies stronger than lower scopes.
- CI and release workflows now run `pnpm lint`.

## Security

- Structured logger metadata redacts sensitive fields.
- Mini App launch grants are signed, expiring, use-limited, and revocable.
- Mini App session tokens are stored as hashes and can be revoked.
- Production Mini App projection APIs fail closed without a valid Mini App bearer session.
- The starter Caddy route exposes only the public Mini App auth/action endpoints and blocks generic `/api/*` access from the public reverse proxy.
- Security docs now define approval defaults, forbidden MVP actions, rotation/revocation, and audit checklist.

## Production routing notes

- Telegram webhook delivery URL: `https://<domain>/telegram/webhook`.
- Telegram Mini App public URL: `https://<domain>/miniapp` or explicit `HAPPYTG_MINIAPP_URL`.
- `HAPPYTG_APP_URL=http://localhost:3001` is development-only and is not used for production `web_app` links when `HAPPYTG_PUBLIC_URL` or `HAPPYTG_MINIAPP_URL` provides a public HTTPS Mini App URL.
- The bot attempts `setChatMenuButton` for the Telegram Menu/Main Mini App when the Mini App URL is valid public HTTPS; `/ready` reports degraded Mini App readiness if the URL is invalid or menu verification fails.

## Upgrade notes

- Update all workspace versions to `0.4.0`.
- Set `HAPPYTG_MINIAPP_LAUNCH_SECRET` explicitly before production use.
- Set `HAPPYTG_MINIAPP_URL=https://<domain>/miniapp` or set `HAPPYTG_PUBLIC_URL=https://<domain>` so HappyTG can derive it.
- In BotFather, set the bot Menu Button/Main Mini App to the same Mini App public URL if automatic setup is disabled or cannot be verified.
- Set `GRAFANA_ADMIN_PASSWORD` before exposing Grafana.
- Keep `/metrics` internal or behind your own access control.
- Back up `.happytg-dev/control-plane.json`, host `~/.happytg/`, `.env`, and repo-local `.agent/tasks/<task-id>/` bundles before rollout.

## Known limitations

- PostgreSQL/NATS durable event migration remains future work; the MVP still uses the existing file-backed control-plane store.
- OTel exporter wiring is documented as an operational target but not fully instrumented in-process in this release.
- Raw git diff collection is still represented through proof artifacts and summary projections, not a live host-synced patch stream.
