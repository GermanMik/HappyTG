# HappyTG 0.4.2

HappyTG `0.4.2` repairs Mini App rendering, local Mini App port drift, and project/session control from the Mini App.

## Highlights

- `0.4.2`
- Mini App pages are served as `text/html`, so Telegram/browser WebViews render sessions, projects, approvals, reports, and task details instead of showing raw HTML source.
- Mini App session cards and session detail pages visibly label Codex CLI sessions.
- Mini App Projects lists host-reported workspaces and can create a Codex CLI session from the selected project.
- Docker Compose separates the host Mini App port from the container listener: host `3007` can map to container `3001` while Caddy keeps using Docker-network upstream `miniapp:3001`.
- Host-run Caddy can set `HAPPYTG_MINIAPP_UPSTREAM=127.0.0.1:3007` when Mini App runs directly on a non-default local port.

## Upgrade Notes

- If local Mini App moved to `3007`, keep the local settings together:

  ```env
  HAPPYTG_MINIAPP_PORT=3007
  HAPPYTG_APP_URL=http://localhost:3007
  HAPPYTG_DEV_CORS_ORIGINS=http://localhost:3007,http://127.0.0.1:3007
  ```

- For Docker Compose Caddy, leave `HAPPYTG_MINIAPP_UPSTREAM` unset.
- For host-run Caddy, set `HAPPYTG_MINIAPP_UPSTREAM=127.0.0.1:3007` or the actual local Mini App listener.
