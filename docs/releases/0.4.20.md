# HappyTG 0.4.20

HappyTG `0.4.20` fixes Telegram Mini App launch probes that could see a `404` before the app opened from a button.

## Highlights

- `0.4.20`
- Mini App now handles `HEAD /` as a valid app entrypoint.
- The response includes `x-happytg-service: miniapp`, matching the service identity expected by launch diagnostics.
- This keeps the existing `GET /miniapp` rendering path unchanged while making button/WebView preflight behavior safe.

## Safety Notes

- Telegram remains a render/approval surface, not the internal transport for agent events.
- No Mini App auth, session, policy, approval, or mutation contracts were relaxed.
- No migration is required.

## Verification

- Added targeted Mini App HTTP regression coverage for `HEAD /`.
- `pnpm release:check --version 0.4.20`
