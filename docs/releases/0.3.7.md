# HappyTG 0.3.7

## Release version

- `0.3.7`

## Why this release

- Remove unwanted prefilled state from the interactive Telegram bot token field so the first paste starts from an empty value instead of appending onto stale draft or `.env` content.
- Lock the release against the observed retry path where users clear the token field and paste again in the same session.

## Fixed

- Interactive installer token capture now ignores persisted `TELEGRAM_BOT_TOKEN` values from existing `.env` files and saved install drafts when the Telegram form is opened.
- Reducer coverage now proves that a token field can be cleared completely and then replaced by a freshly pasted BotFather token without breaking the edit flow.
- Interactive install coverage now verifies the Telegram screen starts with `<required>` for the token field even when prior token state exists, while a newly pasted token still reaches `.env` merge output.

## Verification

- Added regression coverage for:
  - reducer clear-then-paste token replacement
  - interactive install transcript showing a blank initial Telegram token field despite persisted token state
- Release validated with:
  - `pnpm --filter @happytg/bootstrap test`
  - `pnpm --filter @happytg/bootstrap typecheck`
  - `pnpm release:check --version 0.3.7`
  - `pnpm happytg task validate --repo . --task HTG-2026-04-14-release-037-telegram-token-field`

## Upgrade notes

- Update all workspace versions to `0.3.7`.
- Existing `.env` or saved installer draft token values no longer prefill the interactive Telegram token field; enter or paste the token explicitly during interactive install runs.
