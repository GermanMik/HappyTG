# Telegram UX

## Bot Role

Telegram is the command and notification surface for:

- pairing,
- host selection,
- session start,
- approvals,
- concise progress,
- completion summaries.

## Mini App Role

Use the Mini App for:

- diff inspection,
- task bundle browsing,
- long logs,
- verification reports,
- host inventory,
- session and approval history.

## UX Rules

- keep chat messages short and decision-oriented,
- avoid dumping long logs into Telegram,
- escalate to Mini App for details,
- approvals must show action, workspace, risk, and expiry,
- long-running sessions must emit heartbeat summaries without spam.
